# AGENTS.md

Guidance for AI agents (and humans) working in this repo.

## What this is

Personal portfolio for Amrita Vidhate ‚Äî a single **static** page (education,
experience, skills, selected work, contact email). No backend, no framework, no
runtime. Deployed as static files (e.g. Apache from `/var/www/html`).

> An earlier version had an Express/MySQL contact-form backend. It was removed
> on 2026-06-18 (form ‚Üí static email). See `docs/plans/` for the change plans
> and code review.

## Stack

- **HTML + vanilla JavaScript** ‚Äî no framework.
- **Tailwind CSS v4** (`@tailwindcss/cli`) ‚Äî the only build step. Compiles
  `frontend/src/input.css` ‚Üí `frontend/public/output.css`.
- Icons are **inline SVGs** (sourced from Bootstrap Icons, MIT) ‚Äî no icon
  font / CDN. The **Oswald** font is still loaded via CDN in `index.html`.
- Custom `script.js`: starfield canvas, image carousel, scroll/fade animations,
  navbar toggle (replaces Bootstrap's collapse), contact-email tooltip,
  scroll-to-top button, and an **EN/FR language switcher** (client-side i18n).
- **i18n**: English text is authored in `index.html` (the no-JS fallback);
  French lives in the `translations` map in `script.js`. Elements carry
  `data-i18n` (sets textContent) or `data-i18n-content` / `-aria` / `-alt`
  (sets that attribute). Language resolves as localStorage ‚Üí browser language
  (`fr*` ‚Üí French) ‚Üí English, persists the user's explicit choice to
  `localStorage.lang`, and updates `<html lang>`. A tiny inline `<head>` script
  resolves the language before first paint (stamps `data-lang` on `<html>` and,
  for non-English, hides the body via `.i18n-loading`) so French visitors never
  flash English; `script.js` reuses that value and clears `.i18n-loading` once
  the swap is done.

## Build & run

All npm commands run from `frontend/`. Use the `Makefile` from the repo root.

| Task | Make | Raw |
|------|------|-----|
| Install deps | `make install` | `cd frontend && npm install` |
| Watch CSS (dev) | `make dev` | `cd frontend && npm run dev` |
| Build minified CSS | `make build` | `cd frontend && npm run build` |
| Serve the site | `make serve` | `python3 -m http.server --directory frontend/public 8080` |

The site is static ‚Äî once `output.css` exists you can also just open
`frontend/public/index.html` in a browser.

## Conventions

- **Never hand-edit `frontend/public/output.css`** ‚Äî it is generated. Change
  styles via Tailwind utility classes in `index.html` or theme tokens /
  `@keyframes` / `@layer base` in `frontend/src/input.css`.
- Custom design tokens live in the `@theme` block: `--color-primary` (#013ca7),
  `--color-bg` (#121420), `--font-oswald`, card/skill shadows, and the
  `slideInOut` / `fillBar` animations.
- After editing styles, run `make build` (or keep `make dev` running) so
  `output.css` reflects changes.
- Vanilla JS only ‚Äî no bundler, no transpile. Keep `script.js` framework-free.
- When adding/changing copy, update **both** `en` and `fr` in the `translations`
  map in `script.js` and tag the markup with `data-i18n*`. Keep prose that has
  inline markup wrapped in a `<span data-i18n>` so translation values stay plain
  text.
- Images go under `frontend/public/assets/images/`; work screenshots under
  `assets/images/work-snippets/<project>/`.

## Gotchas

- `output.css` is **generated and git-ignored** ‚Äî never edit or commit it.
  Run `make build` to (re)generate it locally before deploying (deploy is
  manual; there is no server-side build step).
- The inline `<head>` language script + its `.i18n-loading` `<style>` exist to
  prevent a flash of English for French visitors ‚Äî don't remove them. They
  mirror `resolveInitialLang()`; if you change the resolution rules, change both.
