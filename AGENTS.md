# AGENTS.md

Guidance for AI agents (and humans) working in this repo.

## What this is

Personal portfolio for Amrita Vidhate — a single **static** page (education,
experience, skills, selected work, contact email). No backend, no framework, no
runtime. Deployed as static files (e.g. Apache from `/var/www/html`).

> An earlier version had an Express/MySQL contact-form backend. It was removed
> on 2026-06-18 (form → static email). See `docs/plans/` for the change plans
> and code review.

## Stack

- **HTML + vanilla JavaScript** — no framework.
- **Tailwind CSS v4** (`@tailwindcss/cli`) — the only build step. Compiles
  `frontend/src/input.css` → `frontend/public/output.css`.
- **Bootstrap Icons** + **Oswald** font, loaded via CDN in `index.html`.
- Custom `script.js`: starfield canvas, image carousel, scroll/fade animations,
  navbar toggle (replaces Bootstrap's collapse), contact-email tooltip.

## Build & run

All npm commands run from `frontend/`. Use the `Makefile` from the repo root.

| Task | Make | Raw |
|------|------|-----|
| Install deps | `make install` | `cd frontend && npm install` |
| Watch CSS (dev) | `make dev` | `cd frontend && npm run dev` |
| Build minified CSS | `make build` | `cd frontend && npm run build` |
| Serve the site | `make serve` | `python3 -m http.server --directory frontend/public 8080` |

The site is static — once `output.css` exists you can also just open
`frontend/public/index.html` in a browser.

## Conventions

- **Never hand-edit `frontend/public/output.css`** — it is generated. Change
  styles via Tailwind utility classes in `index.html` or theme tokens /
  `@keyframes` / `@layer base` in `frontend/src/input.css`.
- Custom design tokens live in the `@theme` block: `--color-primary` (#013ca7),
  `--color-bg` (#121420), `--font-oswald`, card/skill shadows, and the
  `slideInOut` / `fillBar` animations.
- After editing styles, run `make build` (or keep `make dev` running) so
  `output.css` reflects changes.
- Vanilla JS only — no bundler, no transpile. Keep `script.js` framework-free.
- Images go under `frontend/public/assets/images/`; work screenshots under
  `assets/images/work-snippets/<project>/`.

## Gotchas

- `output.css` is **generated and git-ignored** — never edit or commit it.
  Run `make build` to (re)generate it locally before deploying (deploy is
  manual; there is no server-side build step).
